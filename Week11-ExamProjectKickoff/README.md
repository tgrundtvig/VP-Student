# Week 11: Exam Project Kickoff

**Date:** 28 April 2026

## What We Did In Class

No new technical material this week. The session was devoted to **your exam
project**: getting it scoped, designed, and started.

For each student we worked through:
- **Domain choice** — what is your project about?
- **Scope** — what's the smallest version that demonstrates the methodology
  end-to-end? What can be cut?
- **Interface sketch** — what are the 2–4 top-level interfaces?
- **Data shapes** — what's a record? What's an interface?
- **First milestone** — what would "minimum viable" look like?

The pattern is the same one we've used all course:
1. Pick the top-level interfaces.
2. Sketch workflows that use them.
3. Recursively design the next layer.
4. Implement only when you reach trivial leaves.

## For The Exam

The oral exam (9 June) is centred on your project. At this point you should
be able to:
- State your project's purpose in one sentence.
- Sketch the top-level interfaces on a whiteboard from memory.
- Explain *why* each interface exists and *what would break* if you merged
  or split them.
- Show a workflow that uses only interfaces (no concrete classes).
