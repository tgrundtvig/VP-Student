# Week 1: The Naive Approach

Welcome to Week 1! This week, we're going to build a simple text-based game the "normal" way - focusing on getting something working quickly. This sets the foundation for understanding why better design approaches exist.

## Learning Objectives

By the end of this week, you will be able to:

- Review and apply basic Java OOP concepts (classes, objects, methods)
- Design and implement a simple text-based game
- Use basic Java collections (ArrayList, HashMap)
- Write simple methods and class structures
- Experience rapid prototyping and iterative development

## Time Estimate

- **Pre-class work**: 1.5-2 hours
- **Class session**: 3 hours (17:00-20:00)
- **Post-class exercises**: 2-3 hours

## Overview

This week is about **building something that works**. We're not worried about perfect design, comprehensive testing, or future maintainability. The goal is to:

1. Refresh your Java knowledge
2. Build confidence by completing a working program
3. Experience the joy of rapid development
4. Set a baseline for comparison in Week 2

Think of this as "coding like you normally would" - instinctive, practical, get-it-done programming.
